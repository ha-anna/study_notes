this is simply put, a reference to an object. It’s dynamic and depends on how it’s called.

4 Rules to this. How is it called?

  1. in the global context this = Window (global object, undefined in strict mode)
    console.log(this);
    function whatIsThis() {
  console.log(this);
}

whatIsThis();

// ! in strict mode:

function whatIsThis() {
  'use strict';
  console.log(this);
}

whatIsThis(); // logs out null
  1. as a method on an object this = properties of the object (object on left side of dot)
    const user = {
  first: 'Reed',
  last: 'Barger',
  greetUser() {
    console.log(`Hi, ${this.first} ${this.last}`);  
  }  
}

user.greetUser() // logs out Hi, Reed Barger

    nested objects:

    this refers to the element on the immediate left hand side when calling a method

    const userInfo = {
  title: "Programmer",
  user: {
     first: 'Reed',
     last: 'Barger',
     greetUser() {
      console.log(`Hi, ${this.first} ${this.last}`);  
     }   
  }  
}

userInfo.user.greetUser() //logs out Hi, Reed Barger
  1. as a constructor function or class constructor (the instance itself with new)
    function User(first, age) {
  this.first = first;
  this.age = age;
}

User.prototype.getAge = function() {
  console.log(`${this.first}'s age is ${this.age}`);  
}

const user2 = new User('jane', 25);
user2.getAge();
    class User {
  constructor(first, age) {
    this.first = first;
    this.age = age;  
  }  
  
  getAge() {
    console.log(`${this.first}'s age is ${this.age}`);  
  }
}

const user = new User('Bob', 24);
user.getAge();
  1. as a DOM event handler (the element itself)
    1. in addEventListener this will refer to event.target
    const button = document.createElement('button');
button.textContent = "Click";
document.body.appendChild(button);

button.addEventListener('click', function() {
  console.log(this);
})

We can use call(), apply() or bind() to find out what this is:

const user = {
  name: "Reed",
  title: "Programmer"  
}

function printUser() {
  console.log(`${this.name} is a ${this.title}`);
}

printUser.call(user);

By using call() we can call the function as if it was a method on the object.

const user = {
  name: "Reed",
  title: "Programmer"  
}

function printUser() {
  console.log(`${this.name} is a ${this.title}`);
}

printUser.apply(user);

apply() works the same way.

We can also pass additional arguments when using call() and apply().

How the call() and apply() differ? 

const user = {
  name: "Reed",
  title: "Programmer"  
}

function printBio(city, country) {
  console.log(`${this.name} is a ${this.title} in ${city}, ${country}`);
}

printBio.call(user, 'London', 'England')
const user = {
  name: "Reed",
  title: "Programmer"  
}

function printBio(city, country) {
  console.log(`${this.name} is a ${this.title} in ${city}, ${country}`);
}

printBio.apply(user, ['London', 'England'])

When using those we will have to call/apply those every single time, the original function will remain unchanged.

Sometimes, you might need to use a method over and over with the this context of another object. And in that case we need to use another function: bind().

const user = {
  name: "Reed",
  title: "Programmer"  
}

function printUser() {
  console.log(`${this.name} is a ${this.title}`);
}

const userDescription = printUser.bind(user);
userDescription()

What bind() does is that it gives us brand new function with the explicitly bound this, so it’s always going to be bound to the context that we provide to it.

bind() will bind it for good, even if we later try to use it on another object, it will not work.

arrow functions

Reminder: arrow functions don’t have their ownthis.

const user = {
  first: 'Bob',
  fn() {
     console.log(this.first); 
  },
  arrowFn: () => {
    console.log(this.first);  // console.log(user.first);
  }
}

user.arrowFn(); // logs out null

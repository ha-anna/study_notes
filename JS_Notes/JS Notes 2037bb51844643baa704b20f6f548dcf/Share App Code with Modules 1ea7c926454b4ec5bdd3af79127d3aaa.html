Modules (files) allow us to splitting our code into multiple files while still being able to share code between those files. We isolate our JS functionality into separate files and modules help us with keeping each bit of our app clearly defined from each other.

 <script src="index.js" type="module"></script>

Modules can load each other and share code.

Modules are in the strict mode.

Each module has its own top-level scope. Top-level variables and functions from a module aren’t seen in other scripts.

✨ SEPARATIONS OF CONCERNS!!!! ✨

How to check for strict mode?

If we console.log(this) and it returns undefined(null) it means we are in the strict mode.

import { getDate } from './utils/date.js';

  const date = new Date();

Default exports:

Modules can have at most only one default export.

Those are when you only have one thing that you want to make available to the rest of your app.

Here we don’t have to use {} and we can change the name in import however we like (this only works for default!!)

date.js

export default function getDate() {
  const year = new Date().getFullYear();
  const date = `${new Date()}, year: ${year}`;  
  return date;
}

or:

export const add = (a, b) => a + b
export default getDate;

index.js

import getDate from './utils/date.js';

const year = 2019;
class App {
  constructor() {
    this.render()  
  }  
  
  render() {
    document.getElementById('root').innerHTML = `
      <div>Date: ${getDate()}</div>
    `
  }
}

new App()

When default exporting an anonymous function:

export default function () {
// your beautiful code comes next
}

export default () => {
// your beautiful code comes next
}

When this component/function throws you an error, the callstack won't be able to show you the origin of the error, since there is no name added to the function. It's important to give a name to your functions:

export default function App() {
  // your beautiful code comes next
}

export default const App = () => {
  // your beautiful code comes next
}

If you wish you can also change the internal name of your function:

export default function App() {
  // your beautiful code comes next
}
App.name = "Super App"

export default const App = () => {
  // your beautiful code comes next
}
App.name = "Super App"

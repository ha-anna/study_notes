How Maps Can Do What Objects Can’t

ModuleM5Objects & Maps
Reviewed

Object.keys can be used to check what keys are in an object:

const nums = {
  1: 1,
  true: true
};

console.log(Object.keys(nums)) //logs out ["1", "true"]

Maps are Object+ and solve many problems that there were with Objects.

new Map([
	['key', 'value'] // creating first value pair
]);

The Map object holds key-value pairs and remembers the original insertion order of the keys. Any value (both objects and primitive values) may be used as either a key or a value.

MapObject
A Map's keys can be any value (including functions, objects, or any primitive).The keys of an Object must be either a String or a Symbol.
Performs better in scenarios involving frequent additions and removals of key-value pairs.Not optimized for frequent additions and removals of key-value pairs.
They are ordered, arranged in insertion order.Unordered, not arranged in insertion order.

const map1 = new Map([
  [1, 1],
  [true, true]  
]);

To add new pair to an existing map:

map1.set('key', 'value');

[...map1.keys()] 

console.log([...map1.keys()]) // logs out [1, true, "key"]

We can iterate through all of Map data with .forEach(): 

const map1 = new Map([
  [1, 1],
  [true, true]  
]);

map1.set('key', 'value'); //adds values to map

map1.forEach((value, key) => {. //value first!! more logical since we usually want value, not key
  console.log(key, value);  
});

We can use objects in maps:

const user1 = { name: "john" }
const user2 = { name: "mary" }

const secretKey1 = "asldjfalskdjf";
const secretKey2 = "alksdjfakjsdf";

const secretKeyMap = new Map([
  [user1, secretKey1],
  [user2, secretKey2]  
]);

console.log(secretKeyMap)

Accessing values:

const user1 = { name: "john" }
const user2 = { name: "mary" }

const secretKey1 = "asldjfalskdjf";
const secretKey2 = "alksdjfakjsdf";

const secretKeyMap = new Map([
  [user1, secretKey1],
  [user2, secretKey2]  
]);

const key = secretKeyMap.get(user1);
console.log(key); //logs out "asldjfalskdjf"

However, accessing keys becomes harder, and it can take up a lot memory when our object are very long and that will make your app slower.

So in this case it is good to use a variant of map that’s optimized for garbage collection, called WeakMap. It works exactly like Map.

const user1 = { name: "john" }
const user2 = { name: "mary" }

const secretKey1 = "asldjfalskdjf";
const secretKey2 = "alksdjfakjsdf";

const secretKeyMap = new WeakMap([
  [user1, secretKey1],
  [user2, secretKey2]  
]);

const key = secretKeyMap.get(user1);
console.log(key);

For the normal Object there is no way to check the length. We have to do this:

const nums = {
  1: 1,
  true: true
};

Object.keys(nums).length;

However, Map has a convenient alternative:

const userMap = new Map([
  ["name", "john"],
  ["verified", true]  
]);

console.log(userMap.size);

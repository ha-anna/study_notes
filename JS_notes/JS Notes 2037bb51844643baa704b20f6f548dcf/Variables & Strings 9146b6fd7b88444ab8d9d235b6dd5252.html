Variable Basics

Variables are like boxes.

There are three types of variables: var, let, and const.

We use = to assign data to a variable: var username = 'Anna'

We can log out anything in our script by using console.log().

let and const and "strict mode"

→ let can be reassigned after being initialized

let age;  //initialize

age = 26; //declare

age = 36; //reassign

→ const cannot be reassigned after being declared

const age = 26; // must be initialized with a value

age = 36; // throws !TypeError: Assignment to constant variable.

The benefit of using const when declaring is that you make it clear that this variable will stay the same throughout your code. If you only use let and then your code is more than 100 lines, checking if a certain variable was ever reassigned will take time. However, if you made it with const you won’t have to check at all. Using const makes code cleaner and safer.

Hoisting with let and const:

'use strict';

// let / const
console.log(age); //throws !ReferenceError: Cannot access 'age' before initialization
let age = 26;

Naming Variables

There are some standards to naming variables:

  • camelCase is used to name variables
  • names only contain letters, numbers, symbol $ _
  • first character must not be a number
  • words that cannot be used (called reserved words) i.e. var var
  • casing matters (let name and let Name are two different variables)
    const name = 'Jane';
const Name = 'Mark';
const NAME = 'Fred';

console.log(name, Name, NAME); //logs out Jane,"Mark","Fred"
  • variable identifiers should be self-descriptive and shouldn’t require comments for others to know what they hold
    let fullName = `${firstName} ${lastName}`;

let isModalVisible = false; //"is" tells us it is either yes(true) or no(false)

// also thanks to this naming convention it reads like a sentence:

if (isModalVisible) { // if the modal is visible: do something
  // do something  
}

→ The closer that we can get our code to reading like it was just plain instruction, the better.

  • boolean variables: is or has (i.e. isLoading, hasPosition)
  • for variables who’s value should never be changed by other developers manually either we write the variable name in ALLCAPS and separate words with underscores i.e. const COLOR_RED = '#f00';
